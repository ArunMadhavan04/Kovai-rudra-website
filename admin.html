Going live for everyone (Firebase / Supabase)

This panel saves projects to the browser's local storage, so it works instantly with no setup — perfect for managing the gallery on one device. To make changes appear for every visitor on every device, connect a free cloud database. The code is already structured for an easy swap:

  • Create a free Firebase project and enable Firestore + Storage (or use Supabase).
  • Replace the three store.* helpers in admin.js and the loader in main.js with the calls below.
  • Enable Firebase Authentication for real, secure admin login (the access code here is a simple gate, not full security).
// 1) load the SDK in admin.html & index.html
import { initializeApp } from "https://www.gstatic.com/firebasejs/10.x/firebase-app.js";
import { getFirestore, collection, getDocs, addDoc, doc, updateDoc, deleteDoc }
  from "https://www.gstatic.com/firebasejs/10.x/firebase-firestore.js";

const app = initializeApp({ /* your config */ });
const db  = getFirestore(app);

// 2) read projects on the website (main.js)
const snap = await getDocs(collection(db, "projects"));
const projects = snap.docs.map(d => ({ id: d.id, ...d.data() }));

// 3) add / edit / delete in the admin (admin.js)
await addDoc(collection(db, "projects"), project);
await updateDoc(doc(db, "projects", id), project);
await deleteDoc(doc(db, "projects", id));

Upload photos to Firebase Storage and save the returned URL in the project's img field. That's the whole upgrade — the rest of the site stays exactly as it is.
